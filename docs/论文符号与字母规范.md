# 论文符号与字母规范

> 本规范用于组内论文写作、开题/中期报告、组会汇报和投稿前自查。正式投稿时，若目标会议、期刊或模板对符号字体有明确要求，应优先服从目标模板。

符号规范的目标不是“看起来高级”，而是让读者在 Method 和 Experiments 中少猜一次。一个符号第一次出现时必须被定义；同一个符号全文只表示同一种含义；同一类数学对象尽量使用同一类字体。

本规范参考计算机视觉、inverse rendering、image formation、diffusion prior 类论文的常见写法来约定符号。以后写论文时，可以先查这里，再写公式。

---

## 1. 总原则

| 原则 | 具体要求 | 反面问题 |
|---|---|---|
| 第一次出现必须定义 | 符号第一次出现在正文或公式中时，紧跟一句解释 | 读者看到 `x`、`I`、`L`，不知道是像素、图像还是光照 |
| 同一符号全文同义 | `\mathbf{x}` 如果表示 3D point，就不要后文又表示 image feature | Method 看似严谨，实际需要读者反复猜 |
| 同一对象同一字体 | 标量、向量、矩阵、集合、函数、随机变量分别使用固定格式 | 公式里粗体、斜体、花体混用，没有规律 |
| 公式服务于叙事 | 每个公式前说“要解决什么”，公式后说“它说明什么” | 公式堆得很多，但读者不知道为什么需要它 |
| 下标表达角色 | 用下标说明视角、像素、时间、迭代、样本、类别 | 用很多相近字母表示相近概念，增加记忆负担 |
| 符号少而稳定 | 能复用已有符号就不要新造符号 | 一篇文章出现几十个一次性变量 |

## 2. 字体格式与数学对象约定

| 对象类型 | 推荐格式 | 示例 | 含义说明 |
|---|---|---|---|
| 标量 scalar | 小写斜体 | `$x, y, t, \lambda$` | 单个数值，如坐标分量、时间步、权重 |
| 向量 vector | 小写粗体 | `$\mathbf{x}, \mathbf{v}, \mathbf{n}$` | 坐标、方向、法向、特征向量 |
| 矩阵 matrix | 大写粗体 | `$\mathbf{K}, \mathbf{R}, \mathbf{T}$` | 相机内参、旋转矩阵、变换矩阵 |
| 张量 tensor | 粗体大写或花体 | `$\mathbf{I}, \mathcal{F}$` | 图像、特征体、体素网格；图像常写作粗体大写 |
| 集合 set | 花体或大写 | `$\mathcal{D}, \mathcal{S}, \mathcal{P}$` | 数据集、场景集合、像素集合 |
| 函数 function | 正常斜体或罗马体 | `$f_\theta(\cdot), g(\cdot)$` | 神经网络、映射函数、渲染函数 |
| 网络参数 | 希腊字母 | `$\theta, \phi$` | 可学习参数；不同网络用不同参数 |
| 损失函数 | 花体 L | `$\mathcal{L}_{\mathrm{rec}}$` | 总损失和子损失 |
| 概率分布 | 小写 `p` 或 `q` | `$p(\mathbf{x}), q_\phi(\mathbf{z})$` | 数据分布、先验分布、近似后验 |
| 随机变量 | 大写斜体或语境说明 | `$X, Y, Z$` | 若论文概率推导较多，可用大写表示随机变量 |
| 常数 | 普通体或大写 | `$C, \epsilon$` | 固定阈值、常数项、小扰动 |

> 组内默认：向量统一写 `\mathbf{x}`，矩阵写 `\mathbf{K}`，集合写 `\mathcal{D}`，损失写 `\mathcal{L}`。不要一会儿用 `\vec{x}`，一会儿用 `\boldsymbol{x}`，除非全文有明确区分。

## 3. 计算机视觉论文常用符号

### 3.1 图像、像素与相机

| 概念 | 推荐符号 | 解释 | 写作示例 |
|---|---|---|---|
| 输入图像 | `$\mathbf{I}$` | 一张 RGB 图像 | `Given an input image $\mathbf{I}$, ...` |
| 第 `i` 个视角图像 | `$\mathbf{I}_i$` | 多视角中的第 `i` 张图 | `$\{\mathbf{I}_i\}_{i=1}^{N}$ denotes posed images.` |
| 预测图像 | `$\hat{\mathbf{I}}$` | 模型渲染或重建结果 | `We compare $\hat{\mathbf{I}}$ with $\mathbf{I}$.` |
| 像素坐标 | `$\mathbf{u}=(u,v)$` | 2D pixel coordinate | `For each pixel $\mathbf{u}$, we cast a camera ray.` |
| 像素集合 | `$\mathcal{P}$` | 图像中被采样或监督的像素集合 | `The loss is computed over pixels $\mathbf{u}\in\mathcal{P}$.` |
| 相机内参 | `$\mathbf{K}$` | intrinsic matrix | `$\mathbf{K}$ maps camera coordinates to pixels.` |
| 相机位姿 | `$\mathbf{P}_i$` 或 `$(\mathbf{R}_i,\mathbf{t}_i)$` | 第 `i` 个视角的外参 | 多视角论文中二选一，全文保持一致 |
| 相机中心 | `$\mathbf{o}$` | camera origin | 常用于 ray 表示 |
| 射线方向 | `$\mathbf{d}$` | ray direction | 通常为单位向量 |
| 相机射线 | `$\mathbf{r}(s)=\mathbf{o}+s\mathbf{d}$` | 从相机出发的空间射线 | `$s$` 表示 ray depth 或采样距离 |

### 3.2 几何、材质与光照

| 概念 | 推荐符号 | 解释 | 写作示例 |
|---|---|---|---|
| 3D 点 | `$\mathbf{x}\in\mathbb{R}^3$` | 空间点坐标 | `$\mathbf{x}$ denotes a 3D surface point.` |
| 法向 | `$\mathbf{n}$` | surface normal | `$\mathbf{n}$ controls the local shading response.` |
| 深度标量 | `$d$` | 单个像素或点的深度 | 标量深度不用粗体 |
| 深度图 | `$\mathbf{D}$` | 整张 depth map | map / image 用大写粗体 |
| 几何表示 | `$\mathbf{G}$` | geometry representation | 可表示 mesh、SDF、density field 等 |
| 反照率 | `$\boldsymbol{\rho}$` 或 `$\mathbf{A}$` | albedo / diffuse reflectance | 单点材质用 `$\boldsymbol{\rho}$`，整图用 `$\mathbf{A}$` |
| 镜面项 | `$\mathbf{s}$` 或 `$\mathbf{S}$` | specular component | 与 albedo 区分，避免都写成 color |
| 粗糙度 | `$r$` 或 `$\mathbf{R}_{\mathrm{rough}}$` | roughness | 避免和 ray `$\mathbf{r}$` 混淆，必要时用描述性下标 |
| 材质参数 | `$\mathbf{m}$` 或 `$\mathbf{M}$` | roughness、metallic、specular 等组合 | 单点用 `$\mathbf{m}$`，整张 map 用 `$\mathbf{M}$` |
| 光照强度 | `$\ell$` | scalar light intensity | 标量用小写 |
| 光照方向 | `$\mathbf{l}$` | light direction | 方向向量用小写粗体 |
| 环境光表示 | `$\mathbf{L}$` | illumination / environment map | 首次出现必须说明参数化方式 |
| BRDF | `$f_r$` | bidirectional reflectance distribution function | `The outgoing radiance is computed by BRDF $f_r$.` |
| 渲染函数 | `$\mathcal{R}(\cdot)$` | 从几何、材质、光照生成图像 | `$\hat{\mathbf{I}}=\mathcal{R}(\mathbf{G},\mathbf{M},\mathbf{L})$` |

> 重点约定：单个点上的物理量用小写粗体，例如 `$\mathbf{x}, \mathbf{n}, \mathbf{m}$`；整张图或整张 map 用大写粗体，例如 `$\mathbf{I}, \mathbf{A}, \mathbf{S}, \mathbf{D}$`。

### 3.3 神经网络、特征与扩散模型

| 概念 | 推荐符号 | 解释 | 写作示例 |
|---|---|---|---|
| 网络函数 | `$f_\theta$` | 参数为 `$\theta$` 的网络 | `$f_\theta(\mathbf{x})$ predicts material parameters.` |
| 另一个网络 | `$g_\phi$` | 参数为 `$\phi$` 的网络 | 避免多个网络都写成 `$f$` |
| 特征向量 | `$\mathbf{z}$` | latent code / feature vector | `$\mathbf{z}\in\mathbb{R}^C$` |
| 特征图 | `$\mathbf{F}$` | feature map | 大写粗体表示整张 feature map |
| 数据集 | `$\mathcal{D}$` | training dataset | `$\mathcal{D}=\{(\mathbf{I}_i,\mathbf{P}_i)\}_{i=1}^{N}$` |
| 采样噪声 | `$\boldsymbol{\epsilon}$` | diffusion noise | 向量或张量噪声用粗体 |
| 扩散时间步 | `$t$` | diffusion timestep | 标量时间步用小写斜体 |
| 去噪网络 | `$\epsilon_\theta$` 或 `$s_\theta$` | noise predictor / score network | 根据论文路线二选一 |
| 先验分布 | `$p(\mathbf{x})$` | data prior / material prior | 概率分布用小写 `p` |
| 近似分布 | `$q(\mathbf{x})$` | approximate posterior 或 forward process | 需要在首次出现时说明 |

## 4. 公式写法规范

公式不要孤零零出现。推荐使用“三段式”：先说公式目的，再给公式，再解释每个关键符号。

| 步骤 | 应该写什么 | 示例 |
|---|---|---|
| 公式前 | 说明为什么需要这个公式 | `To regularize material estimation under unknown illumination, we introduce a diffusion prior over albedo maps.` |
| 公式中 | 保持符号稳定、下标清楚 | `$\mathcal{L}=\mathcal{L}_{\mathrm{rec}}+\lambda_{\mathrm{prior}}\mathcal{L}_{\mathrm{prior}}$` |
| 公式后 | 解释每一项的作用 | `$\mathcal{L}_{\mathrm{rec}}$ enforces image reconstruction, while $\mathcal{L}_{\mathrm{prior}}$ constrains material plausibility.` |

### 4.1 损失函数推荐写法

```latex
\mathcal{L}
= \mathcal{L}_{\mathrm{rec}}
+ \lambda_{\mathrm{prior}}\mathcal{L}_{\mathrm{prior}}
+ \lambda_{\mathrm{smooth}}\mathcal{L}_{\mathrm{smooth}}.
```

| 符号 | 含义 | 写作建议 |
|---|---|---|
| `$\mathcal{L}$` | 总损失 | 总损失统一用花体 L |
| `$\mathcal{L}_{\mathrm{rec}}$` | 重建损失 | 下标用 roman text，不写成斜体单词 |
| `$\lambda_{\mathrm{prior}}$` | prior loss 权重 | 权重统一用 `\lambda`，下标说明作用 |
| `$\lambda_{\mathrm{smooth}}$` | smoothness loss 权重 | 不要用 `a, b, c` 这类无含义字母 |

### 4.2 优化问题推荐写法

```latex
\theta^\ast
= \arg\min_{\theta}
\sum_{i=1}^{N}
\mathcal{L}\big(\hat{\mathbf{I}}_i(\theta), \mathbf{I}_i\big).
```

这类公式要明确三件事：优化变量是谁，训练样本从哪里来，损失比较的是预测量和观测量中的哪两个对象。

### 4.3 预测值、真值和中间量

| 类型 | 推荐写法 | 示例 | 说明 |
|---|---|---|---|
| 预测值 | 加 hat | `$\hat{\mathbf{I}}, \hat{\mathbf{A}}, \hat{\mathbf{n}}$` | 模型输出或估计值 |
| 真值 | 上标 star 或 `gt` | `$\mathbf{I}^\ast, \mathbf{A}^{\mathrm{gt}}$` | ground truth；全文二选一 |
| 输入观测 | 不加 hat | `$\mathbf{I}$` | 表示真实输入或观测 |
| 误差项 | `\Delta` | `$\Delta\mathbf{I}=\hat{\mathbf{I}}-\mathbf{I}$` | 残差或差异 |
| 中间变量 | 加语义下标 | `$\mathbf{F}_{\mathrm{mat}}$` | 不要写成无意义的 `$\mathbf{F}_1$` |

## 5. 推荐符号表模板

论文中符号较多时，建议在 Method 开头或 appendix 放一个 notation table。组内写作时至少在草稿中保留符号表，方便自查。

| 符号 | 类型 | 含义 | 备注 |
|---|---|---|---|
| `$\mathbf{I}_i$` | 图像 | 第 `i` 个视角的输入图像 | 多视角输入 |
| `$\mathbf{u}$` | 向量 | 像素坐标 | `$\mathbf{u}\in\mathbb{R}^2$` |
| `$\mathbf{x}$` | 向量 | 3D 表面点 | `$\mathbf{x}\in\mathbb{R}^3$` |
| `$\mathbf{n}$` | 向量 | 表面法向 | 单位向量 |
| `$\mathbf{A}$` | 图像 / map | albedo map | 整张反照率图 |
| `$\mathbf{S}$` | 图像 / map | specular map | 整张镜面项图 |
| `$\mathbf{L}$` | 环境量 | illumination / lighting representation | 首次出现需说明具体参数化方式 |
| `$\mathcal{R}$` | 函数 | differentiable renderer | 输入几何、材质、光照，输出渲染图 |
| `$\theta$` | 参数 | 待学习网络参数 | 与 `$\phi$` 区分不同网络 |
| `$\mathcal{D}$` | 集合 | 训练数据集 | `$\mathcal{D}=\{(\mathbf{I}_i,\mathbf{P}_i)\}_{i=1}^{N}$` |

## 6. 最终检查清单

- 是否所有符号第一次出现时都被定义。
- 是否同一个符号全文只表示一个含义。
- 是否标量、向量、矩阵、集合、损失函数使用了稳定字体。
- 是否区分了单点物理量和整张 map。
- 是否区分了预测值 `$\hat{\cdot}$`、真实值 `$\cdot^\ast$` 或 `$\cdot^{\mathrm{gt}}$`。
- 是否每个下标都有含义，例如视角 `i`、像素 `\mathbf{u}`、迭代步 `t`、样本数 `N`。
- 是否损失函数的每个子项都解释了作用，而不是只列公式。
- 是否避免了用 `A/B/C` 或 `x/y/z` 随便命名重要模块。
- 是否图注、正文、公式里的符号保持一致。
- 是否在投稿前按目标模板检查了粗体、斜体、希腊字母、上下标显示是否正常。

## 7. 组内推荐默认写法

| 写作场景 | 推荐写法 |
|---|---|
| Method 中第一次引入变量 | `Let $\mathbf{x}\in\mathbb{R}^3$ denote a 3D surface point.` |
| 引入多视角输入 | `Given posed images $\{\mathbf{I}_i, \mathbf{P}_i\}_{i=1}^{N}$, ...` |
| 引入渲染过程 | `$\hat{\mathbf{I}}=\mathcal{R}(\mathbf{G},\mathbf{M},\mathbf{L})$` |
| 引入材质先验 | `We use a prior $p(\mathbf{A})$ to regularize plausible albedo maps.` |
| 引入总损失 | `The final objective combines reconstruction, prior, and smoothness terms.` |
| 汇报或草稿中暂定符号 | 在符号后写中文解释，不要只放公式 |

> 最重要的一句话：符号不是装饰，而是论文逻辑的一部分。符号越稳定，读者越容易把注意力放在你的方法和贡献上。
